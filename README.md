# CS3B_3_sotuken2026

全体開発スケジュール（ロードマップ）
🏁 第1目標：手動登録・表示機能の完成
「ユーザーが自分で入力し、それがDBに保存され、一覧で見られる」状態。

【フェーズ1：環境構築と土台作り】
目的: 全員が開発に着手できるインフラと共通ルールを整える。

インフラ: Amplifyプロジェクト初期化、GitHub連携、RDS（PostgreSQL）の構築。

バックエンド: DBテーブル作成、APIの「型（JSON形式）」を定義しフロントと合意。

フロントエンド: React/Expoプロジェクト作成、Tailwind CSS導入、共通デザイン設定。

共通: テスト用のダミーデータ（5件）を定義し、開発の基準にする。

【フェーズ2：基本機能（CRUD）の実装】
目的: データの「読み書き」を完成させ、アプリとしての最小機能を動かす。

バックエンド: サブスクの「登録(POST)」「一覧取得(GET)」APIを実装。

フロントエンド: 手動登録用フォームの作成、メインホーム画面（一覧表示）の構築。

インフラ: フロントとバックをAmplify上で接続し、実際にデータが保存されるか検証。

🏁 第2目標：自動取得機能と可視化の実装
「メールから情報を抜き出し、それをグラフやカレンダーで表現する」状態。

【フェーズ3：自動取得（Gmail解析）の開発】
目的: アプリの核心である「三井住友カード」の明細解析を成功させる。

バックエンド: Gmail APIとの連携、三井住友カード明細メールの正規表現解析ロジック実装。

バックエンド: 解析したデータをDBへ自動保存する処理の統合。

インフラ: Lambdaの実行時間（タイムアウト）調整、メール取得の定期実行設定。

【フェーズ4：データの可視化とUI改善】
目的: 溜まったデータをユーザーが直感的に理解できるようにする。

フロントエンド: グラフライブラリによる支出推移の表示、カレンダー画面の実装。

フロントエンド: 登録済みデザイン案を忠実に再現するブラッシュアップ作業。

🏁 最終フェーズ：公開準備と認証
「セキュリティを固め、自分以外のユーザーも使えるようにする」状態。

【フェーズ5：認証基盤の統合と総仕上げ】
目的: Amazon Cognitoを導入し、個人のデータを守る。

インフラ: Amplify Auth (Cognito) の有効化。

フロントエンド: ログイン・サインアップ画面の実装と、メイン画面への統合。

バックエンド: 認証トークン（JWT）を確認し、自分のデータのみ返却する権限管理の実装。

全体: バグ修正、動作テスト、デプロイ完了。

📅 開発の優先順位（依存関係）
[最優先] インフラのAmplify初期化 ＆ APIの型定義（これがないと誰も動けない）

[並行] 手動登録画面作成 ＆ 三井住友カード解析の研究（解析は時間がかかるため早めに着手）

[中間] 手動登録のAPI連携（ここでアプリが形になる）

[後半] グラフ・カレンダー表示（データが必要なため）

[最終] 認証機能（開発中は認証があると面倒なため、最後に入れる）
